# 문서 우선(Docs-First) 규칙

## 핵심 원칙
- 이 레포는 **문서 우선** 방식으로 관리합니다.
- MQTT 토픽, JSON payload, UI 동작, Arduino 펌웨어 동작은 `docs/` 문서를 기준으로 합니다.
- 요구사항이 바뀌면 **해당 문서를 먼저 수정**한 뒤 코드를 맞춥니다.

## 문서에 없는 새 요소 추가 시
- 새 토픽, 새 키, 새 UI 요소를 추가해야 한다면 **먼저 관련 문서를 갱신**합니다.
- 문서 변경 후 코드를 수정합니다.

## 비기능 규칙
- `gh1`, `gh2`를 소스 코드로 분기 복제하지 말고 **설정 또는 파라미터**로 처리합니다.
- 가능한 한 **단순한 구조**로 구현합니다.
